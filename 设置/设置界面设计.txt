设置界面概念图

Windows 11 风格桌面应用设置页面，中文界面设计，整体延续首页、便签页与回忆书的统一产品语言。界面尺寸约 1440x900px，采用纯净浅色主题、低饱和灰阶、充足留白、极细分割线、无阴影常驻内容块与内容优先的信息组织方式。视觉参考 Apple Settings、Linear Settings、Vercel Dashboard、Notion Preferences 与前面主页面的 AI Workspace 风格。

整体视觉原则：

* Minimal、Calm、Premium、Productivity
* 不使用深色侧边栏，不使用高饱和色块
* 不使用玻璃拟态、渐变背景或复杂装饰
* 设置项以清晰文本、轻量输入控件和细分割线组织
* 卡片只用于聚合相关设置，不做厚重面板
* 所有修改即时保存，不需要全局“保存”按钮
* Hover、选中、成功和错误反馈均保持低存在感

全局应用骨架：

最左侧保留与首页、便签页、回忆书一致的极简图标侧边栏，宽度约 80px，背景 #FCFCFD。侧边栏仅显示图标，不显示 Logo、产品名称、新建按钮、导航文字、用户头像或昵称。顶部依次放置首页、便签、回忆本三个线性图标；底部仅保留设置图标。当前设置页中，设置图标使用极浅灰蓝背景与深色图标强调。图标尺寸约 16px，点击区域约 40x40px，圆角 12px。

设置页主体采用“全局侧边栏 + 设置二级导航 + 设置内容区”的布局。全局侧边栏固定 80px；设置二级导航宽度约 220px；右侧内容区占据剩余空间。整体背景为 #FCFCFD，二级导航背景可使用 #F8FAFC，内容区以 #FFFFFF 为主。边界使用 #E5E7EB / #EEF2F7 的极细分割线。

字体与色彩：

* 字体采用 Inter / SF Pro Display / PingFang SC
* 页面标题 #0F172A，16-18px，600
* 分组标题 #0F172A，13-14px，600
* 设置项主文本 #1E293B，13-14px，500
* 设置项说明 #64748B / #94A3B8，12-13px
* 输入框文字 #0F172A，占位文字 #94A3B8
* 主要按钮使用 #0F172A 黑色胶囊或圆角按钮
* 普通按钮、标签和状态使用 Slate 灰阶
* 成功状态使用低饱和绿色，错误状态使用低饱和红色，不做大面积警示色
* 供应商、默认模型、快捷键、关于等设置子页面的字号必须与偏好设置保持一致：分组标题约 15px，行内容和控件文字约 13px，辅助说明约 12px；控件高度、卡片内边距和行距也要保持紧凑，不使用大号列表项或大号输入框

顶部与内容节奏：

设置页面不设置独立顶部标题栏，不显示额外的页面标题、说明文字或全局操作按钮。右侧内容区直接从顶部留白开始，默认留白约 28-32px。当前页面名称通过二级导航选中态与内容卡片标题共同表达，避免重复信息。供应商、统计等页面的操作入口放在各自内容模块内部，而不是放在全局 header 中。

一、设置二级导航

二级导航位于全局图标栏右侧，宽度约 220px，采用纵向列表，顶部显示小标题「设置」，字号 13px，字重 600，颜色 #0F172A。下方依次展示：

* 偏好设置
* 供应商
* 默认模型
* 快捷键
* 统计
* 关于

导航项高度约 36px，圆角 10-12px，左右 padding 12px。当前页面使用 #F1F5F9 或 #EEF2F7 背景、深色文字和稍高字重；未选中项为 #64748B，hover 时出现 #F8FAFC / #F1F5F9 轻微背景。导航项可使用极简线性图标，但图标尺寸不能超过 15px，整体不能喧宾夺主。

二、偏好设置页面

偏好设置作为默认页面。右侧内容区必须在可用区域内水平居中，不贴左侧二级导航。内容最大宽度约 1080px，顶部留白 28-32px，整体采用纵向宽卡片布局。每个分组为白色轻卡片，圆角约 18px，边框 #E5E7EB / #EEF2F7，不使用阴影。卡片之间间距 16-20px。

个人信息模块：

模块标题为「个人信息」，采用类似系统设置页的行式布局。卡片内不堆叠大段说明文字，每行左侧为 13px 设置项名称，右侧为对应控件，行高约 58px，行与行之间使用 #EEF2F7 极细分割线。包含以下设置项：

* 每日工作时长：右侧为数字输入框，单位小时，默认 8
* 日薪：右侧为金额输入框，人民币符号「¥」作为输入框外侧后缀显示，不放在输入框内部
* 所在行业：右侧为下拉选择或可搜索输入，用于优化 AI 总结和行业建议

字体与显示模块：

模块标题为「字体与显示」。同样采用行式设置列表，控件靠右对齐。包含以下设置项：

* 应用字体：右侧显示「系统默认」下拉按钮，并提供独立重置图标按钮
* 字体大小：右侧为数字输入框，默认 100，百分号「%」作为输入框外侧后缀显示，不提供重置按钮

应用字体设置项点击后打开字体选择弹窗。用户完成选择后立即生效并自动保存，不需要确认按钮。

字体选择弹窗：

* 弹窗宽度约 520px，高度约 620px
* 背景 #FFFFFF，圆角 24px，边框 #E5E7EB，阴影柔和
* 顶部标题「应用字体」，右上角关闭按钮
* 搜索框提示文字「输入以过滤字体...」
* 列表顶部固定「系统默认」选项
* 字体列表读取系统已安装字体
* 每个字体项包含字体名称和实时预览「Aa字」
* 当前选中项使用极浅灰蓝背景和右侧 check 图标
* 列表支持滚动，滚动条保持低存在感

选择“系统默认”时，应用跟随操作系统字体：Windows 使用 Segoe UI Variable，macOS 使用 SF Pro，Linux 使用 Noto Sans。字体选择框右侧的重置按钮用于清除自定义字体并立即恢复系统默认。

行为与启动模块：

模块标题为「行为与启动」。包含两个开关：

* 开机自启动：控制应用是否随系统启动自动运行
* 显示更新：控制启动时是否自动检测并提示更新

开关采用 iOS / Apple Settings 风格，小尺寸圆角滑块。开启状态使用 #0F172A 或低饱和深色，关闭状态为 #E5E7EB。所有设置切换后即时保存。

组件设置模块：

模块标题为「组件设置」。先只保留一个开关：

* 显示桌面组件：控制桌面组件是否显示

该模块与偏好设置内其他分组保持一致，使用白色圆角卡片、细边框、58px 行高和小尺寸开关。所有设置切换后即时保存。

三、供应商页面

供应商页面用于管理 AI 服务供应商、接口配置和模型资源。页面在设置二级导航右侧采用两栏布局：左侧为供应商列表，右侧为当前供应商详情。整体接近系统偏好设置中的资源管理面板，不再使用厚重卡片堆叠。背景以 #FFFFFF 为主，左右两栏之间使用 #E5E7EB / #EEF2F7 极细分割线。

供应商列表区域：

左侧供应商列表宽度约 320-340px，顶部为搜索框，提示「搜索供应商或分组」，高度约 40px，圆角 14-16px，背景 #F1F5F9 / #F3F4F6，不使用明显边框。列表展示所有供应商配置，例如 OpenAI、Google、Gemini、OpenRouter、DeepSeek、本地模型服务和兼容 OpenAI API 的第三方服务。

每个供应商项展示：

* 极简服务商图标、首字母标识或低饱和彩色圆形标识
* 供应商名称
* 右侧状态标签：启用 / 禁用

列表项高度约 44px，圆角约 14-16px。选中项使用 #F1F5F9 / #F3F4F6 背景，未选中项 hover 时使用极浅灰背景。状态标签使用低饱和绿色和橙色：启用为 #DCFCE7 / #16A34A，禁用为 #FFEDD5 / #F97316。列表底部固定“+ 添加”按钮，按钮为白底细边框，圆角约 14-16px，文字 15px 600，不使用黑色主按钮，以保持列表工具属性。

添加供应商弹窗：

点击添加后打开模态弹窗。弹窗宽度约 760px，圆角 22px，白色背景，作为悬浮窗允许使用柔和阴影。顶部左侧标题「添加供应商」，字号约 18px，右上角关闭按钮。标题下方为三等分模板选择区，仅提供 OpenAI、Google、Claude 三个入口。模板按钮高度约 40px，当前选中模板使用浅灰实心圆角块，未选中项只显示文字，hover 时出现极浅灰背景。

基础配置项包含：

* 是否启用
* 名称
* API Key，默认密码隐藏
* Base URL
* API 路径

弹窗不提供额外 API 类型切换配置项。是否启用使用一整行浅灰背景开关项，高度约 58px，开启态为黑色轨道与白色滑块。名称、API Key、Base URL、API 路径按单列纵向排列，输入框高度约 44px，圆角约 12px，浅灰背景，边框 #CBD5E1 / #D1D5DB。弹窗底部右侧提供黑色「+ 添加」主按钮，高度约 44px，不需要左侧保存提示。点击添加后校验配置，创建成功后自动加入供应商列表并切换到详情页。

供应商详情区域：

右侧详情区占据剩余空间，内容最大宽度约 1080-1120px，并在右侧区域内水平居中。顶部第一行只显示当前供应商名称，例如「OpenAI」、一个设置图标按钮以及右上角启用开关，不出现额外页面标题。该行高度约 44px，底部保留极细分割线。供应商启用开关使用黑色开启态和白色滑块，关闭态使用浅灰背景。

基础配置区域不使用外层卡片。表单直接排布在右侧内容区中，每个字段采用“上方标签 + 下方输入框”的结构，形成轻量编辑页面：

* API Key：输入框默认密码隐藏，右侧提供 eye 图标显示/隐藏；标签行右侧提供「测试」操作，使用心跳/连接状态类线性图标和文字
* API Base URL：单行输入框，示例值 `http://127.0.0.1:1234/v1`
* API 路径：单行输入框，示例值 `/chat/completions`

输入框高度约 48px，圆角约 14px，背景 #F8FAFC / #F9FAFB，边框 #E5E7EB，文字 15px。focus 时仅轻微加深边框，不使用发光描边。所有修改实时保存。

模型管理区域位于 API 配置下方。标题行左侧显示「模型」与数量圆形徽标，右侧是一组轻量图标操作：搜索、批量选择、添加、删除、刷新，以及文字按钮「获取」。图标尺寸约 16px，按钮 hover 仅显示极浅灰底。

模型列表采用单个轻量折叠组，而不是多张卡片。外层为 16px 圆角细边框容器，顶部组标题行显示下拉箭头和模型分组名，例如「qwen」。下方模型行展示模型标识符，例如 `qwen/qwen3-1.7b`，左侧有极简模型图标，右侧提供高级配置、能力配置、参数设置和移除等图标操作。整体行高约 52-56px，分割线极细，避免传统表格感。

模型操作包括：

* 设为默认模型
* 编辑模型参数
* 高级配置
* 删除模型

高级配置支持上下文长度、最大输出长度、温度参数、视觉能力、多模态能力、推理能力等。高级配置可以使用抽屉或弹窗呈现，避免在详情页中堆叠过多字段。

四、默认模型页面

默认模型页面用于配置应用内三个核心业务场景使用的模型。布局采用单列大卡片，不需要三栏。内容区最大宽度约 1120-1180px，并在右侧区域水平居中。顶部仅显示轻量标题「默认模型」，不设置额外 header。

每个模型类型使用一张白色圆角卡片，圆角约 22px，边框 #E5E7EB，不使用阴影。卡片顶部左侧为线性图标、模型类型标题和一句说明，右侧可显示设置图标按钮。卡片底部为灰色圆角选择条，高度约 56px，背景 #F1F5F9，左侧使用圆形状态徽标，例如「已」或「未」，右侧显示当前选择的模型。

模型分组仅保留以下三项：

* 智能生成模型：用于首页输入后的智能整理、总结与结构化生成
* 编辑补全模型：用于便签页的续写、润色、改写与编辑补全
* 回忆书模型：用于回忆书问答、记忆检索与个人记录整理

模型选择条点击后进入模型选择或配置流程。若模型不可用，使用低对比度状态提示，不使用错误色大面积提示。所有修改即时保存。

五、快捷键页面

快捷键页面采用系统设置式行列表，不使用传统表格。内容区最大宽度约 1120-1180px。顶部左侧显示轻量标题「快捷键」，标题字号、字重、颜色以及标题到第一项的间距与默认模型页面保持一致，不需要刷新或批量操作按钮。主体为一张白色圆角卡片，圆角约 20-22px，边框 #E5E7EB。

快捷键配置只保留一项：「显示/隐藏页面」。单行高度约 58-60px，左侧为功能名称，中间为快捷键输入框，右侧依次为重置图标、清除图标和启用开关。快捷键输入框高度约 40px，宽度约 280px，圆角 12px，边框 #CBD5E1，背景 #F8FAFC。未设置时显示「按下快捷键」，已设置时显示用户录入的组合键。开关采用黑色开启态和白色滑块。

点击快捷键输入框后进入录制状态。若快捷键冲突，行内以低饱和红色文字提示冲突功能，避免弹窗打断。重置按钮恢复默认快捷键，清除按钮移除当前快捷键。

六、统计页面

统计页面用于展示应用使用情况、模型调用数据和长期趋势。页面采用纵向布局，整体视觉与首页收益卡片和热力图保持一致。

时间筛选区域：

顶部使用胶囊按钮展示：

* 全部
* 最近30天
* 上个月
* 上个季度
* 自定义

默认选中「最近30天」。切换后，页面内热力图、总览卡片和趋势图同步刷新。按钮选中态使用 #0F172A 深色文字和 #F1F5F9 背景，避免高饱和。

自定义时间段弹窗：

点击自定义后打开轻量弹窗。开始时间和结束时间输入框不可直接手输日期，点击后弹出日期选择器。日期选择器采用月历视图，支持月份切换、跨月跨季度跨年度选择。

日期约束规则：

* 先选开始日期后，结束日期自动禁用所有早于开始日期的日期
* 先选结束日期后，开始日期自动禁用所有晚于结束日期的日期
* 禁用日期降低透明度并禁止点击
* 由于选择阶段已限制非法范围，点击应用时无需额外报错

该方案遵循“限制错误输入优于事后提示错误”的原则。

聊天热力图模块：

时间筛选下方展示聊天热力图，视觉参考首页的 GitHub 风格热力图。这里记录用户 commits 次数或核心交互活跃度，时间范围必须是精确的一年，例如当前日期为 2026-06-17 时，热力图从 2025-06-17 展示到 2026-06-17；不要为了凑满整周而展示 2025-06-12 这类额外日期。若需要星期行对齐，只能使用不可见占位格，不可显示范围外日期。格子数量比首页更多但仍保持连续紧密的小方块矩阵，而不是拉大间距形成稀疏图。热力图外围需要保留月份标识和星期标识，顶部月份单行显示，例如「7月」「8月」「9月」，并与对应月份起始列对齐；左侧显示「一 / 三 / 五」。色阶必须 100% 参考首页热力图配色，依次使用 #F1F5F9、#DCFCE7、#BBF7D0、#86EFAC、#4ADE80。Hover 显示与首页一致的白色 tooltip，包含日期和次数。热力图 tooltip 必须使用全局 fixed 浮层实现，并根据视口边界自动调整位置，不能被热力图横向滚动容器或卡片边界裁剪。

总览统计模块：

热力图下方展示核心指标卡片。卡片使用白色背景、20px 圆角和细边框，不使用阴影。指标包括：

* 总结数：用于统计用户在首页使用智能生成的数量
* 编辑补全次数：用于统计用户在便签页面触发编辑补全的次数
* 总记录数：用于统计日报、周报、月报数量
* 日报数：用于统计用户日报数量
* 周报数：用于统计用户周报数量
* 月报数：用于统计用户月报数量
* 输入 Tokens：用于统计用户向模型发送的全部 Token 数量，反映输入内容规模以及资源消耗情况
* 输出 Tokens：用于统计模型生成内容所消耗的 Token 数量，用于衡量 AI 实际输出规模
* 缓存 Tokens：用于统计命中缓存机制所节省或使用的 Token 数量，帮助用户了解缓存带来的成本优化效果
* 应用启动次数：用于统计应用被启动和打开的总次数，用于反映产品使用频率以及用户活跃程度

卡片强调数字，说明文字后退。所有指标随当前时间范围实时刷新。

用量趋势模块：

页面底部展示不同模型供应商或模型来源的使用变化趋势。该模块默认展示最近一个月，按照当前筛选范围按天展示，不按周、月或季度聚合；当用户切换到很长的范围，例如 800 多天时，单日槽宽需要自动缩小，并通过横向滚动浏览完整时间轴。图表视觉参考轻量产品数据面板：左上仅保留标题「用量趋势」，主体使用大量留白，不显示厚重坐标轴、网格线或摘要卡。底部使用一排浅灰短虚线表示日粒度基线，只有发生用量的日期才出现细圆角堆叠柱。柱体在短范围内宽度约 16-18px，长范围内可压缩到约 4-8px；柱体圆角不能做成上下 50% 的胶囊形，应使用更扁平的小圆角，例如 4px 左右。Hover 时柱体不需要上移，而是在柱体外出现一个略大一圈的空心描边框包裹当前柱体，不设置默认常驻 tooltip。图表区域仅在内容横向溢出时才拦截鼠标滚轮并转换为 scrollLeft 左右移动；未溢出时不阻止默认滚动，页面继续正常上下滚动。长范围默认打开时滚动到最新日期。

供应商图例固定放在图表底部，使用小色块和名称横向排列，支持横向滚动以容纳几十到几百个供应商。供应商颜色不能使用固定有限色表，优先根据供应商稳定 ID 或名称哈希动态生成；推荐使用现代清爽色板：以蓝、青、薄荷、紫、玫瑰、珊瑚等方向为主，保持足够明快但不过度饱和，避免复古黄棕、土橙、灰旧色和高饱和亮色，也不要做成彩虹图表。Hover 提示卡使用浅灰白背景、12px 圆角和柔和阴影，宽度约 200-220px，字号约 11px，顶部日期约 13px；下方按供应商列出彩色圆点、供应商名称和原始 token 数值，例如 `272475 tokens`；没有 hover 时不显示提示卡。用量趋势 tooltip 必须使用全局 fixed 浮层实现，并根据视口边界自动调整位置，不能被横向滚动容器、卡片边界或页面滚动容器裁剪。
